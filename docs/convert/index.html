Recover your ModHeader profiles

Pull your profiles out of ModHeader and into Header Handler. Everything runs in your browser — nothing you drop here is ever uploaded.

Heads up: Google and Microsoft removed ModHeader from their stores in July 2026 after build 7.0.18 was found bundling a hidden data collector. Chrome and Edge force-disable it, so its own Export button may no longer open. Don't try to re-run it — the steps below get your profiles out without ModHeader working.

ModHeader's profiles are still saved on disk until you uninstall it. This reads them straight from the browser's data folder — the extension never has to run.

  1. Paste this into TerminalPowerShell and run it. It reads ModHeader's saved profiles from Chrome, Edge and Brave (and their variants) and saves them to modheader-dump.txt:
    It only reads files — ModHeader never runs. It uses only built-in tools (nothing to install). The dump can contain header values like auth tokens, so keep it local and delete it when you're done. If iCloud "Desktop & Documents" sync is on, your Desktop syncs to iCloud — delete modheader-dump.txt once you've imported. Saved 0 bytes? Fully quit the browser that had ModHeader and run it again.

No terminal? Find the folder by hand
  1. Open the browser's User Data folder — Chrome shown here:
    In Finder press ⇧⌘G and paste it. Library is hidden by default. Paste it into the File Explorer address bar. AppData is hidden by default.
  2. Search it for the id — there's no single Default folder to rely on (Chrome names profiles Default, Profile 1, Profile 2…), so search rather than guess:
    idgpnmonknjnojddfkpgkljpfnnfcklj
    Type it into the File Explorer search box (top-right). In the Finder window press ⌘F, scope to this folder, search the name. You want the <id> folder inside Local Extension Settings (or Sync / Managed Extension Settings). Ignore the IndexedDB one named …_idgp…_0.indexeddb.leveldb — that's the pulled build's data cache, not your profiles.
